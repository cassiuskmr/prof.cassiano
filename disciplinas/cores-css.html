Por onde começar

Decimal está em todo lugar no seu dia: o preço de um produto, a hora no relógio, a velocidade no painel do carro. A tela do computador usa esse mesmo decimal — só que para misturar luz em vez de contar coisas.

Cada ponto da tela (pixel) é formado por três luzinhas: uma vermelha, uma verde e uma azul. A intensidade de cada uma é um número decimal de 0 a 255. Combinando os três números, você obtém qualquer cor — esse é o sistema RGB (Red, Green, Blue).

Analogia · três torneiras de tinta luminosa

Imagine três torneiras — uma de luz vermelha, uma verde e uma azul — cada uma com um registro que vai de 0 (fechado) a 255 (totalmente aberto). Abrir as três no máximo dá branco. Fechar as três dá preto. Qualquer outra combinação dá uma cor diferente. É exatamente assim que cada pixel da sua tela funciona.

O problema é que escrever três números decimais (tipo 51, 102, 204) é longo para usar em código. O CSS resolve isso comprimindo os três números em um único código hexadecimal — a mesma base 16 que você já viu na aula de Sistemas de Numeração. Vamos destrinchar essa conversão.

01

RGB — três números decimais

Cada canal de cor é um número decimal entre 0 e 255. Por que justo até 255? Porque cada canal é guardado em exatamente 1 byte — 8 bits.

Lembrando do byte

8 bits formam 1 byte, com 256 valores possíveis: de 00000000 (0) a 11111111 (255). É exatamente a faixa de cada canal de cor — nada de coincidência, é a mesma matemática da aula de Sistemas de Numeração.

Misture os três canais abaixo e observe a amostra de cor mudando ao vivo. Repare que, conforme você arrasta cada controle, o valor decimal e o par hexadecimal daquele canal aparecem lado a lado — é a mesma conversão decimal → hexadecimal de sempre, só que aplicada a uma cor.

02

De decimal para hexadecimal

A conversão é a mesma de dividir por 16, repetidas vezes — só que feita uma vez para cada canal.

Pegue o canal azul do exemplo acima: decimal 204. Dividindo por 16 sucessivamente:

DivisãoResultadoResto
204 ÷ 161212 = C
12 ÷ 16012 = C

Lendo de baixo para cima: CC. Como todo canal de cor vai de 0 a 255, ele sempre vira exatamente 2 dígitos hexadecimais — nunca mais, nunca menos.

Por que sempre 2 dígitos por canal

2 dígitos hexadecimais cobrem de 00 a FF — exatamente 256 valores, igual à faixa decimal 0–255 de cada canal. Três canais × 2 dígitos = os 6 dígitos de um código #RRGGBB.

O atalho de 3 dígitos

Quando os dois dígitos de cada par são iguais, o CSS aceita uma forma curta: #FF6600 pode ser escrito #F60. Repare que isso só funciona quando dá pra repetir o dígito — #3366CC não tem atalho, porque 33, 66 e CC já são pares repetidos! Esse aliás é exatamente o caso do azul do misturador acima.

Volte ao misturador da seção anterior e arraste os controles — a coluna hexadecimal de cada canal se recalcula instantaneamente, sempre por essa mesma divisão por 16.
